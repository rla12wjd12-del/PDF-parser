# Role: 시니어 데이터 엔지니어 및 PDF 파싱 아키텍트 (pdfplumber, 정규식 고도화)

## 1. 작업 배경 및 목표
현재 '건설기술인 경력증명서' PDF를 파싱하여 JSON으로 변환하는 파이썬 코드를 운영 중입니다. `pdfplumber`를 이용해 표 추출과 텍스트 기반 추출을 혼합(하이브리드)하여 사용하고 있으나, 양식이 조금씩 다른 여러 PDF에서 지속적으로 파싱 오류(데이터 누락, 텍스트 밀림, 오인식 등)가 발생하고 있습니다.
제공된 전체 소스코드와 여러 샘플 PDF, 그리고 발견된 오류 내역을 바탕으로 코드의 근본적인 문제점을 분석하고, 파싱 과정 전체를 견고하게 리팩토링하고자 합니다.

- **샘플 PDF 및 결과:** [오류가 발생하는 샘플 PDF 파일들과 기대하는/현재의 JSON 결과 첨부]

  - [예시 1: A 샘플에서 제2쪽 기술경력의 '사업명'이 인식되지 않고 누락됨]
  - [예시 2: B 샘플에서 제1쪽 상훈 표의 행이 밀려서 JSON 키값이 어긋남]

## 3. 단계별 수행 지시사항 (반드시 아래 Step 순서대로 사고하고 답변할 것)

### Step 1: 현재 코드 아키텍처 및 로직 진단 (Static Analysis)
현재 코드(`page_2_parser.py`의 `_parse_flow_blocks`, `section_parsers.py` 등)의 구조를 분석해줘. 
표 추출 실패 시 텍스트로 폴백하는 과정이나, 정규식을 통한 행 분리 로직에서 어떤 한계점이나 설계상 결함이 있어서 현재의 오류들이 발생하는지 논리적으로 진단해줘.

### Step 2: 다중 샘플 교차 검증 (Cross-Validation)
제공된 샘플 PDF들의 양식을 대조해줘. 
에러가 나는 샘플과 나지 않는 샘플 간에 어떤 시각적/구조적 차이(예: 숨겨진 선, 텍스트 줄바꿈 패턴 차이, 페이지 넘김 시 레이아웃 등)가 있는지 분석하고, 현재 코드의 어느 함수가 이 차이를 감당하지 못해 터지는지 짚어줘.

### Step 3: 리팩토링 및 수정 전략 제시 (Strategy)
Step 1, 2의 분석을 바탕으로, 땜질식(Hardcoding) 처방이 아닌 근본적인 해결책(전략)을 제시해줘.
- 예: `pdfplumber`의 `table_settings` 교정, Bounding Box 기반 좌표 추출 도입, 혹은 텍스트 파싱 시 기준점(Anchor) 재설정 등
- **주의:** 코드를 바로 작성하지 말고, 구체적인 수정 전략과 접근법을 먼저 설명해줘. 내가 이 전략을 읽고 컨펌(진행 승인)을 하면 그때 코드를 작성해.

### Step 4: 코드 수정 (승인 대기)
(이 단계는 내가 Step 3의 전략을 '진행해'라고 말한 이후에 수행할 것)
- 합의된 전략을 바탕으로 기존 모듈화된 구조를 해치지 않는 선에서 코드를 리팩토링해줘.
- 어느 파일의 어떤 함수를 교체해야 하는지 명시하고, 수정된 부분에는 주석(`// FIX: [수정 사유]`)을 반드시 달아줘.

### Step 5: 다중 샘플 자동 검증 스크립트 작성
- 코드 수정이 완료되면, 여러 개의 PDF 파일을 한 번에 돌려서 파싱 결과(JSON)에 필수 키값이 누락되었거나 데이터가 밀렸는지 자동으로 대조하고 리포팅해 주는 `test_validator.py` 스크립트를 작성해줘.
