## 파트 1: [관리자용] 현재까지 진행 현황 요약 (비개발자용)

### 프로젝트 목표(요청사항 기준)
- **건설기술인 PDF 경력증명서**를 읽어서 **구조화된 JSON으로 파싱**하고, 이를 **엑셀(.xlsx)로 변환**해 **데이터를 관리/검증**할 수 있게 만드는 프로젝트입니다.

### 한 줄 비유로 설명
- **“경력증명서 PDF를 자동으로 칸칸이 정리해서 JSON/엑셀로 내보내고, 누락/불일치 의심을 자동 경보로 알려주는 정리 도구”**입니다.  
  문서 레이아웃 편차(줄바꿈 깨짐/표 추출 실패 등)에 대비해 **보정 규칙(휴리스틱) + 폴백 추출(텍스트/좌표 기반)**이 포함돼 있습니다.

### 현재 진행도(추정)
- **엔드투엔드( PDF → JSON → Excel ) 파이프라인은 운영 가능한 수준으로 구현된 상태**입니다.
- 다만 “양식 변경/대량 처리/검증 노이즈” 같은 운영 리스크를 줄이기 위한 **안정화 작업(P0/P1)**이 남아 있습니다.

### 현재 제공 가능한 주요 기능(사용자 관점)
- **PDF 입력**(지원 포맷: `.pdf`)을 받아 아래 정보를 추출합니다.
  - 인적사항/등급/자격/학력/교육훈련/상훈/벌점·제재/근무처
  - 기술경력
  - 건설사업관리·감리경력(CM)
  - 요약 인정일수(공사종류별/직무전문분야별)
- 결과를 **JSON(`json_output/`)**과 **엑셀(`excel_output/`)**로 저장합니다.
- 파싱 결과에 대해 **자동 검증(원본 대비 건수/필드 품질)**을 수행하고, 이상 징후는 `"_파싱오류"`로 누적 기록합니다.

### 현재 산출물(예: 샘플/배치 결과)
- **JSON 출력**: `json_output/`에 **40개** 생성(예: `신복수_20260420.json`, `이상인_20260420_6.json` 등)
- **엑셀 출력**: `excel_output/`에 **26개** 생성(예: `신복수_20260420.xlsx`, `이상인_20260420_6.xlsx` 등)
- **테스트 PDF**: `testpdf/`에 **13개** 보유(다양한 날짜/레이아웃 변형 포함)

### 현 상황의 리스크(무엇이 우려되는지)
- **문서 양식 변경 리스크**: 요약 섹션(인정일수) 파서가 현재는 “요약 시작부터 문서 끝까지” 합쳐 읽는 구조라 **부록/추가 페이지 혼입 시 오탐** 위험이 있습니다.
- **대량 배치 성능/안정성**: 처리 과정에서 PDF를 여러 번 여는 경로가 존재해 **IO 비용 증가 및 실패 지점 증가** 가능성이 있습니다.
- **검증 노이즈**: 일부 섹션의 “원본 기대 건수” 산출이 정규식/레이아웃에 의존해 **거짓 불일치 경보**가 발생할 수 있습니다.
- **유지보수 난이도**: 기술경력 파서가 크고 복잡해, 작은 수정도 영향 범위가 커질 수 있습니다.

## 파트 2: [개발자용] 현재 구현 상태 및 기술 진단

### 코드/폴더 구조(핵심만)
- 오케스트레이션/출력
  - `main.py`: 섹션 시작 페이지 탐지 → 파서 호출/병합 → JSON/Excel 저장 → 검증/오류기록
  - `excel_export.py`: JSON(dict) → xlsx 변환(섹션별 시트, `_파싱오류` 시트)
- 파싱 코어
  - `parsers/page_1_parser.py`, `parsers/section_parsers.py`: 1쪽 계열(인적/등급/자격/학력/교육훈련/상훈/벌점/근무처)
  - `parsers/page_2_parser.py`: 기술경력(핵심, 대형 모듈)
  - `parsers/page_3_parser.py`: 건설사업관리·감리경력(CM)
  - `parsers/page_summary_parser.py`: 요약 인정일수
  - `parsers/layout_extractor.py`: PyMuPDF 우선 / pdfplumber 폴백 + 좌표 기반 라인 재구성
- 검증/리포팅
  - `verify_parsing.py`, `scripts/verify/test_validator.py`: 배치 파싱 품질지표/불일치/오염 검출

### 기능별 구현 현황(요약)
| 기능 | 상태 | 비고 |
|---|---:|---|
| PDF→JSON 파이프라인 | 구현됨 | 섹션 탐지/병합/저장/검증 포함 |
| JSON→Excel 변환 | 구현됨 | 섹션별 시트 + 오류 시트 |
| 1쪽 계열 섹션 파싱 | 구현됨 | 텍스트 기반 + 표 기반 보강 |
| 기술경력 파싱 | 구현됨(복잡) | 휴리스틱/폴백 다수, 리팩토링 여지 큼 |
| CM(감리경력) 파싱 | 구현됨 | 표+텍스트 병합, 최근 1년 통계 반영 |
| 요약 인정일수 파싱 | 구현됨(리스크) | 범위 과다(EOF까지 병합) 개선 필요 |
| 배치 검증/리포팅 | 구현됨 | 운영 기준치(FAIL/WARN) 튜닝 여지 |
| 배포/재현성(의존성 고정) | 미흡 | `requirements.txt`/`pyproject.toml` 부재 |

## 파트 3: 리스크/개선 우선순위 (현 시점 기준)

### P0 (즉시 리스크 저감)
- **요약 파서 안전화**: `summary_start..EOF` 병합 완화(종료 앵커 + 최대 병합 페이지 상한)

- **의존성 명시**: `requirements.txt` 또는 `pyproject.toml` 추가(환경 재현성 확보)

- 중복 정규화 로직(회사/근무처/기관명 등)을 공통 유틸로 추출
